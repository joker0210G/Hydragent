# The Library Knowledge Graph Architecture

Imagine a vast library. This library is your entire knowledge base—everything you've saved, thought about, or discussed. 
We're building an architecture that works exactly like this library, where information isn't locked away in rigid boxes, but interconnected in a fluid, non-linear way.

## 📖 The Core Analogy (The Graph Nodes)

1. **The Library (The Whole Graph)**: The entire database containing all your information. Everything exists within the library.
2. **The Shelves (Categories & Genres)**: Divisions based on your preferences. Unlike a real shelf, a **Shelf** here is a fluid concept. A shelf can be "Romance" or "Comedy". Since things are interconnected, the "Romedy" genre can be linked to both the Romance and Comedy shelves.
3. **The Books (Projects & Contexts)**: A Book is like a specific project or room where you have a conversation about a particular topic. A single Book can sit on multiple Shelves (e.g., a story with a mixture of emotions).
4. **The Pages (Sessions & Threads)**: The individual interactions, sessions, or specific pieces of context. Pages are subsets of Books, containing the essence of the conversation. Crucially, the exact same Page can belong to multiple Books if it's relevant to different projects.
5. **The Desk (The Workspace)**: A specific place where you sit down with the Librarian to have a general conversation. If you discuss a new idea on a blank page at the Desk, you can later sort that page into a suitable Book or Shelf.
6. **The Librarian (The AI / LLM)**: Your intelligent assistant who helps you organize, retrieve, and discuss anything in the library.

---

## 🕸️ The Interconnected Web (Why Graph?)

The most important rule of our Library is that **everything is interconnected**. 
- A Book can sit on multiple Shelves.
- A Page can be ripped out and placed into multiple Books.
- The Library itself connects to everything.

To make searching incredibly fast and optimized (saving LLM tokens and reducing tool calls), we cannot use a traditional step-by-step sequential search. Instead, we are using a **Graph Database structure**. In a graph, everything is connected by webs (edges), so the Librarian can instantly pull a Book, look at its connected Pages, and find exactly what you need without reading the entire Library.

## 🗺️ Integrating `graphify` for Knowledge Mapping

To bring this web of knowledge to life, we will integrate **`safishamsi/graphify`**, a Python package designed to extract concepts and build knowledge graphs from raw files.

### How we will use `graphify`:
1. **Download & Integration**: We'll install `graphifyy` (its PyPI package name) into our Python adapter.
2. **Custom Modifications**: We will modify how it's used so it understands our specific *Library, Shelf, Book, Page* structure.
3. **Graph Generation**: It will analyze our data and generate a `graph.json` (the persistent graph) and a visual `graph.html` so you can actually see your interconnected Library.

## 🛠️ Step-by-Step Implementation

### Step 1: The Graph Database Schema
We will refactor our SQLite database to act as the Library's foundation using a generic Node/Edge model:
- **Nodes Table**: Stores every Shelf, Book, Page, and the Desk.
- **Edges Table**: Stores the connections (e.g., this Page *belongs_to* this Book, this Book *sits_on* this Shelf).

### Step 2: The Rust Event Bus Updates
Update the core engine to support Graph queries:
- Add `library.create_node` to make a new Shelf, Book, or Page.
- Add `library.link` to connect a Page to multiple Books or a Book to multiple Shelves.
- Add `library.search` to let the Librarian traverse the graph edges instantly to find context for the Desk.

### Step 3: Python Adapter & Graphify Integration
- Install `graphifyy` in `requirements.txt`.
- Create a new background process that runs `graphify` on the database/files to generate the interactive knowledge map.
- Whenever a general conversation happens at the **Desk**, we will use the LLM to decide if the new "Page" should be linked to an existing Book, a Shelf, or kept at the Desk.
